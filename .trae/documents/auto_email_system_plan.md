# 自动邮件发送系统设计计划

## 项目目标
设计并实现一个能够自动从金山文档读取"3 期需求进展"sheet 页内容，并保留所有格式发送邮件的自动化系统。

## 需求分析
1. **数据源**: 金山文档 (https://www.kdocs.cn/l/coVkqRucbSuv)
2. **目标内容**: "3 期需求进展" sheet 页
3. **关键要求**: 保留所有格式（表格样式、字体格式等）
4. **接收邮箱**: wenspock@gmail.com
5. **发送频率**: 需要支持定时自动发送

## 技术挑战
1. 金山文档可能需要登录认证才能访问
2. 需要解析在线表格并保留格式信息
3. HTML 邮件需要内联 CSS 样式以保持格式
4. Gmail 发送需要配置 SMTP 或 API

## 实现步骤

### 1. 环境准备与依赖安装
- 安装必要的 Python 库：
  - `requests` / `httpx`: HTTP 请求
  - `selenium` / `playwright`: 处理需要登录的网页（如需要）
  - `pandas`: 数据处理
  - `openpyxl`: Excel 文件处理
  - `yagmail` / `smtplib`: 邮件发送
  - `jinja2`: HTML 模板渲染
  - `python-dotenv`: 环境变量管理
  - `schedule` / `APScheduler`: 定时任务

### 2. 文档内容获取模块
- 实现金山文档内容抓取功能
- 处理可能的登录认证（Cookie/Session）
- 解析"3 期需求进展" sheet 页
- 导出为结构化数据（保留格式信息）

### 3. 邮件内容生成模块
- 创建 HTML 邮件模板
- 将表格数据转换为带样式的 HTML 表格
- 内联 CSS 样式以确保邮件客户端兼容
- 支持添加邮件主题、正文等

### 4. 邮件发送模块
- 配置 Gmail SMTP 设置
- 实现邮件发送功能
- 添加错误处理和重试机制
- 支持附件（可选）

### 5. 配置管理模块
- 创建配置文件（.env）
- 管理敏感信息（邮箱密码、API 密钥等）
- 配置发送频率和接收人列表

### 6. 定时任务模块
- 实现定时调度功能
- 支持 cron 表达式或简单间隔
- 添加任务日志记录

### 7. 日志与错误处理
- 实现完整的日志记录
- 添加异常处理和通知机制
- 记录发送历史和状态

### 8. 测试与验证
- 测试文档内容获取
- 测试邮件发送功能
- 验证格式保留效果
- 测试定时任务执行

## 项目文件结构
```
daily_repory_analysis/
├── email_sender/
│   ├── __init__.py
│   ├── config.py          # 配置管理
│   ├── kdocs_fetcher.py   # 金山文档抓取
│   ├── email_generator.py # 邮件内容生成
│   ├── email_sender.py    # 邮件发送
│   └── scheduler.py       # 定时任务
├── templates/
│   └── email_template.html # 邮件模板
├── logs/                   # 日志目录
├── .env                    # 环境变量配置
├── requirements.txt        # 依赖列表
└── main.py                 # 主程序入口
```

## 安全注意事项
1. 使用环境变量存储敏感信息（邮箱密码、API 密钥）
2. Gmail 需要使用应用专用密码（App Password）
3. 不要将 .env 文件提交到版本控制
4. 添加 IP 白名单或访问限制（如部署到服务器）
