# GitHub AI Trending 首页功能 Spec

## Why

当前系统已生成多篇日报、周报和月报，但缺少统一的导航入口和全局视图。用户需要：
- 快速访问最新报告
- 了解整体数据概况
- 发现热门项目和趋势

## What Changes

- **新增**：首页 HTML 文件 (`reports/index.html`)
- **新增**：首页样式文件 (`public/css/index.css`)
- **新增**：首页数据生成脚本 (`scripts/generate-index.js`)
- **修改**：主生成脚本 (`scripts/generate-all.js`) - 添加首页生成调用

**设计原则**：
- 保持与日报/周报一致的深色主题风格
- 简洁实用优先，后续迭代增强
- 纯静态 HTML，无需后端服务

## Impact

- **影响文件**：
  - `reports/index.html` (新增)
  - `public/css/index.css` (新增)
  - `scripts/generate-index.js` (新增)
  - `scripts/generate-all.js` (修改)
  - `package.json` (可能需添加脚本命令)

- **数据依赖**：
  - 从现有日报/周报/月报 HTML 中提取元数据
  - 或从 data/briefs 目录读取 JSON 数据

## ADDED Requirements

### Requirement 1: 全局统计卡片

系统 SHALL 在首页顶部显示全局统计信息：

#### 场景：展示累计报告数量
- **WHEN** 用户打开首页
- **THEN** 显示 6 个统计卡片：
  - 累计日报数量
  - 累计周报数量
  - 累计月报数量
  - 追踪项目总数
  - AI 项目占比 (%)
  - 平均 Stars 数

### Requirement 2: 报告导航模块

系统 SHALL 提供三大报告的快速导航入口：

#### 场景：最新日报入口
- **WHEN** 用户查看首页
- **THEN** 显示最新日报卡片，包含：
  - 日期
  - 上榜项目数
  - AI 项目数
  - 平均 Stars
  - "查看详情"按钮链接到日报 HTML

#### 场景：最新周报入口
- **WHEN** 用户查看首页
- **THEN** 显示最新周报卡片，包含：
  - 周号
  - 上榜项目数
  - 平均 Stars
  - 周度主题（如有）
  - "查看详情"按钮链接到周报 HTML

#### 场景：最新月报入口
- **WHEN** 用户查看首页
- **THEN** 显示最新月报卡片，包含：
  - 月份
  - 上榜项目数
  - 平均 Stars
  - 月度亮点
  - "查看详情"按钮链接到月报 HTML

### Requirement 3: 明星项目榜单

系统 SHALL 展示 Top 10 热门项目：

#### 场景：项目榜单展示
- **WHEN** 用户查看首页
- **THEN** 显示按总 Stars 排序的 Top 10 项目，每项包含：
  - 排名
  - 项目名称（可点击链接）
  - 总 Stars 数
  - 项目类型标签
  - 编程语言
  - 简短描述

### Requirement 4: 基础搜索功能

系统 SHALL 提供简单的项目搜索：

#### 场景：搜索项目
- **WHEN** 用户在搜索框输入关键词
- **THEN** 实时过滤明星项目榜单，显示匹配的项目
- **匹配范围**：项目名称、描述、标签

### Requirement 5: AI 洞察模块（简化版）

系统 SHALL 展示简化的 AI 洞察：

#### 场景：显示热门趋势
- **WHEN** 用户查看首页
- **THEN** 显示：
  - 热门项目类型分布（如 Agent 系统占比）
  - 热门编程语言分布
  - 本周/月最热项目

### Requirement 6: 趋势图表占位

系统 SHALL 为未来趋势图表预留位置：

#### 场景：图表占位区域
- **WHEN** 用户查看首页
- **THEN** 显示"趋势分析"区域，标注"即将推出"
- **目的**：为后续迭代预留空间

## 视觉设计规范

### 颜色主题
- 背景色：`#0d1117` (与日报/周报一致)
- 卡片背景：`#161b22`
- 主文字：`#c9d1d9`
- 次要文字：`#8b949e`
- 强调色：`#58a6ff`
- 边框：`#30363d`

### 布局结构
```
┌─────────────────────────────────────┐
│          顶部标题区                  │
├─────────────────────────────────────┤
│      全局统计卡片 (6 个)             │
├─────────────────────────────────────┤
│    报告导航 (日报/周报/月报)         │
├─────────────────────────────────────┤
│    AI 洞察 + 趋势图表占位            │
├─────────────────────────────────────┤
│    明星项目榜单 (Top 10)             │
├─────────────────────────────────────┤
│    搜索功能（悬浮或固定位置）         │
└─────────────────────────────────────┘
```

### 响应式设计
- 桌面端：完整布局
- 平板端：统计卡片 2 列，报告导航 1 列
- 移动端：所有模块单列显示

## 数据生成策略

### 方案选择：静态数据嵌入

**原因**：
- 当前报告数量有限（<20 篇）
- 纯静态 HTML，加载速度快
- 无需额外数据文件

**实现方式**：
- 在生成首页时，扫描 `reports/` 目录
- 解析现有报告 HTML 的 `<title>` 和统计信息
- 或使用 data/briefs 中的 JSON 数据
- 生成包含完整数据的静态 HTML

## 非功能性要求

### 性能
- 首页加载时间 < 2 秒
- 搜索响应时间 < 100ms

### 兼容性
- 支持现代浏览器（Chrome/Firefox/Safari/Edge）
- 移动端友好

### 可维护性
- 代码结构清晰，注释充分
- 样式与现有系统一致
- 易于后续扩展

## 未来扩展点

1. **高级搜索**：支持多条件筛选
2. **趋势图表**：集成 Chart.js 或类似库
3. **时间轴视图**：展示报告历史
4. **个性化功能**：收藏、订阅等
5. **数据导出**：导出为 PDF/CSV
