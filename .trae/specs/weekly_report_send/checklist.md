## 一、功能验收

### 1.1 飞书版消息

- [ ] **消息结构完整**
  - [ ] Header 包含标题、副标题、模板颜色
  - [ ] 包含本周核心主题模块
  - [ ] 包含关键热点模块
  - [ ] 包含重点技术突破模块
  - [ ] 包含短期趋势预判模块
  - [ ] 包含新兴领域模块
  - [ ] 包含行动建议模块
  - [ ] 包含查看完整报告按钮
  - [ ] 包含页脚时间信息

- [ ] **内容正确性**
  - [ ] 周日期范围计算正确（如 "2026 年 3 月 9 日 - 2026 年 3 月 15 日"）
  - [ ] 周数提取正确（如 "11"）
  - [ ] 所有文本内容与洞察数据一致
  - [ ] GitHub 链接格式正确（`https://github.com/owner/repo`）
  - [ ] 报告链接格式正确（`https://report.wenspock.site/reports/weekly/weekly-2026-W11.html`）
  - [ ] 生成时间格式正确（`YYYY-MM-DD HH:mm`）

- [ ] **交互功能**
  - [ ] 所有链接可点击
  - [ ] 按钮可点击并跳转到正确 URL
  - [ ] 消息在飞书桌面端显示正常
  - [ ] 消息在飞书移动端显示正常

- [ ] **边界情况处理**
  - [ ] highlights 不足 3 个时正常显示
  - [ ] trends 为空时显示"暂无数据"
  - [ ] emergingFields 为空时显示"暂无数据"
  - [ ] insights 完全缺失时降级显示

### 1.2 WeLink 版消息

- [ ] **消息结构完整**
  - [ ] 包含标题（📊 GitHub 趋势周报 · 第 N 周）
  - [ ] 包含日期范围（短格式）
  - [ ] 包含一句话主题
  - [ ] 包含 3 个关键信号
  - [ ] 包含开发者关注领域
  - [ ] 包含建议
  - [ ] 包含报告链接
  - [ ] 包含生成时间（短格式）

- [ ] **字数控制**
  - [ ] 总字数 ≤ 350 字
  - [ ] 标题约 15 字
  - [ ] 主题约 50 字
  - [ ] 每个信号约 30 字
  - [ ] 领域 + 建议约 80 字
  - [ ] 链接约 50 字
  - [ ] 时间约 10 字

- [ ] **内容质量**
  - [ ] 3 个信号提取自 highlights 和 trends
  - [ ] 信号内容精简但不失核心信息
  - [ ] 领域描述清晰易懂
  - [ ] 建议具有可操作性

- [ ] **显示效果**
  - [ ] 在 WeLink 移动端显示完整
  - [ ] 段落间距合理
  - [ ] Emoji 显示正常
  - [ ] 链接可点击

### 1.3 降级模式

- [ ] **触发条件**
  - [ ] insights 文件不存在时自动触发
  - [ ] insights 字段缺失时部分降级

- [ ] **降级内容**
  - [ ] weeklyTheme.oneLiner 使用项目数量生成
  - [ ] weeklyTheme.detailed 显示提示信息
  - [ ] highlights 使用前 3 个项目
  - [ ] topProjects 使用前 1 个项目
  - [ ] recommendations 显示默认提示

- [ ] **降级效果**
  - [ ] 飞书版消息仍然完整可读
  - [ ] WeLink 版消息仍然有价值
  - [ ] 明确提示用户洞察数据暂不可用

## 三、性能验收

### 3.1 响应时间

- [ ] **模板生成**
  - [ ] 飞书版生成时间 < 50ms
  - [ ] WeLink 版生成时间 < 50ms
  - [ ] 总生成时间 < 100ms

- [ ] **消息发送**
  - [ ] 飞书发送时间 < 3s
  - [ ] WeLink 发送时间 < 3s
  - [ ] 双平台总时间 < 5s

- [ ] **整体流程**
  - [ ] 周报生成 + 推送总时间 < 10s
  - [ ] 无阻塞操作
  - [ ] 异步操作合理

### 3.2 资源使用

- [ ] **内存使用**
  - [ ] 无内存泄漏
  - [ ] 峰值内存 < 100MB
  - [ ] 垃圾回收正常

- [ ] **CPU 使用**
  - [ ] 无 CPU 密集操作
  - [ ] CPU 使用率正常
  - [ ] 无死循环

### 3.3 并发处理

- [ ] **并发发送**
  - [ ] 支持同时发送多个周报（不同周）
  - [ ] 无资源竞争
  - [ ] 无数据污染

- [ ] **重试机制**
  - [ ] 发送失败自动重试（最多 3 次）
  - [ ] 重试间隔合理（如 1s、3s、5s）
  - [ ] 重试失败后记录错误
